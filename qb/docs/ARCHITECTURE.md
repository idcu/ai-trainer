# 架构设计文档

## 1. 概述

本文档描述了人工智能训练师题库系统的架构设计，包括技术选型、模块设计、数据流和关键设计决策。

## 2. 技术栈

### 2.1 后端技术栈
- **Web 框架**: Flask 3.0
- **数据库**: SQLite
- **部署方式**: Docker、WSGI、静态文件部署

### 2.2 前端技术栈
- **核心语言**: JavaScript (ES6+)
- **样式**: CSS3
- **模板**: HTML5
- **数据存储**: LocalStorage
- **构建工具**: 原生（无需构建）

## 4. 前端模块设计

### 4.1 主应用 (app-v2.js)

`QuizApp` 类是整个应用的核心入口，负责：
- 初始化所有功能模块
- 管理应用状态
- 协调模块间通信
- 处理用户交互

```javascript
class QuizApp {
    constructor() {
        this.engine = new QuizEngine();      // 答题引擎
        this.ui = new UIRenderer();           // UI渲染器
        this.wrongBook = new WrongBook();     // 错题本
        this.favorites = new Favorites();     // 收藏夹
        this.history = new History();         // 历史记录
        // ... 其他模块
    }
}
```

### 4.2 功能模块 (modules/)

#### 4.2.1 QuizEngine (quiz-engine.js)
- **职责**: 管理答题流程
- **功能**:
  - 加载和管理题目
  - 记录用户答案
  - 计算得分
  - 处理答题进度

#### 4.2.2 UIRenderer (ui-renderer.js)
- **职责**: 负责所有 UI 渲染
- **功能**:
  - 渲染题目和选项
  - 更新进度和得分
  - 渲染各种列表（错题、收藏、历史等）
  - 处理 DOM 操作优化

#### 4.2.3 WrongBook (wrong-book.js)
- **职责**: 错题本管理
- **功能**:
  - 添加错题
  - 删除错题
  - 获取错题列表
  - 错题统计

#### 4.2.4 Favorites (favorites.js)
- **职责**: 收藏夹管理
- **功能**:
  - 收藏/取消收藏
  - 获取收藏列表
  - 检查是否已收藏

#### 4.2.5 History (history.js)
- **职责**: 答题历史管理
- **功能**:
  - 记录答题历史
  - 获取历史记录
  - 历史统计
  - 删除记录

#### 4.2.6 SearchFilter (search-filter.js)
- **职责**: 搜索和筛选
- **功能**:
  - 关键词搜索
  - 题型筛选
  - 结果排序

#### 4.2.7 ProgressSaver (progress-saver.js)
- **职责**: 答题进度保存
- **功能**:
  - 自动保存进度
  - 恢复进度
  - 进度管理

#### 4.2.8 DataIO (data-io.js)
- **职责**: 数据导入导出
- **功能**:
  - 数据导出
  - 数据导入
  - 数据清空

#### 4.2.9 Statistics (statistics.js)
- **职责**: 统计分析
- **功能**:
  - 学习概览
  - 性能评估
  - 学习建议

### 4.3 工具模块 (utils/)

#### 4.3.1 Helpers (helpers.js)
- **职责**: 通用工具函数
- **功能**:
  - 防抖 (debounce)
  - 节流 (throttle)
  - 记忆化 (memoize)
  - 数组打乱
  - 日期格式化
  - 深度克隆

#### 4.3.2 Storage (storage.js)
- **职责**: LocalStorage 封装
- **功能**:
  - 内存缓存
  - 批量写入
  - 数据持久化

#### 4.3.3 DataLoader (data-loader.js)
- **职责**: 数据加载
- **功能**:
  - 加载 JSON 数据
  - 数据缓存
  - 缓存管理

## 5. 数据流

### 5.1 答题流程

```
用户选择题型
    │
    ▼
加载题目 (DataLoader)
    │
    ▼
初始化答题引擎 (QuizEngine)
    │
    ▼
显示答题界面 (UIRenderer)
    │
    ▼
用户答题
    │
    ├─── 答错 → 添加到错题本 (WrongBook)
    │
    ▼
更新进度和得分
    │
    ▼
下一题 / 交卷
    │
    ▼
保存到历史记录 (History)
    │
    ▼
显示结果
```

### 5.2 数据持久化

```
用户操作
    │
    ▼
模块更新数据
    │
    ▼
Storage.set()
    │
    ├─── 更新内存缓存
    │
    └─── 批量写入 LocalStorage (防抖)
            │
            ▼
        beforeunload 事件
            │
            ▼
        强制同步所有待写入
```

## 6. 关键设计决策

### 6.1 纯前端优先设计

**决策**: 优先实现纯前端版本，支持静态部署

**理由**:
- 简化部署流程
- 降低服务器成本
- 支持 GitHub Pages/Gitee Pages
- 离线可用

### 6.2 模块化架构

**决策**: 采用类和模块的方式组织代码

**理由**:
- 提高代码可维护性
- 便于功能扩展
- 代码复用性好
- 便于测试

### 6.3 LocalStorage 数据持久化

**决策**: 使用 LocalStorage 存储用户数据

**理由**:
- 无需后端支持
- 纯前端可用
- 浏览器原生支持
- 数据存储在用户本地

### 6.4 性能优化策略

**决策**: 多层缓存和优化策略

**理由**:
- 减少网络请求
- 优化 DOM 操作
- 减少 I/O 操作
- 提升用户体验

## 7. 扩展性设计

### 7.1 模块扩展

新功能模块应该：
1. 继承或遵循现有模块的设计模式
2. 通过 QuizApp 进行协调
3. 使用统一的数据存储接口
4. 提供清晰的 API

### 7.2 数据源扩展

支持多种数据源：
- JSON 文件
- SQLite 数据库
- REST API
- 其他第三方数据源
